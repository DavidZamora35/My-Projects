Dataset and Algorithms Selection

The dataset we chose from Kaggle was "Airline Passenger Satisfaction"

https://www.kaggle.com/datasets/teejmahal20/airline-passenger-satisfaction

This dataset contains columns with numerical data for: ID, Age, Flight Distance, Departure Delay, Arrival Delay, and many more for various ratings given by passengers, such as Cleanliness and Service (from a scale of 0-5).

It also contains categorical data for: Gender, Customer Type, Type of Travel, Class, and Satisfcation.

All combined, the dataset contains 25 features. To easily view how these features interact with each other, we can encode the categoricals and create a heatmap of the entire dataset, as such:

Decision Tree Classifier

Decision Tree Classifier is an effective algorithm for both numerical and categorical data. It uses a tree structure where each non-leaf node represents a test on an attribute, each branch denotes an outcome of the test, and each leaf node holds a class label. Attribute values of a given sample are tested against the decision tree to predict its class. Because it can work effectively with datasets that contain both numerical and categorical data its likely to be a good choice for this dataset.

KNN Classifier

KNN is a simple algorithm for use in both classification and regression. KNN is capable of accurately predicting data by looking at a data points k nearest neighbors, wherein k is the number of data points it looks at. KNN is a non-parametric instance-based learning algorithm that waits to generalize data until the prediction phase. KNN works best with smaller feature sets and with numeric data, which makes it fairly effective for this dataset as it primarily contains customer ratings ranging from 1 to 5. By providing these ratings as training data, KNN could effectively and accurately predict a passengers overall satisfaction. We will choose the k value based on the result that has the least amount of error, in this case, 3.

Analysis

Accuracy is a measure of the overall correctness of predictions made by the model, and is measured by the the ratio of correctly predicted instances to the total number of instances in the dataset.

F1 Score is the harmonic mean of both precision and recall. Precision is the ratio of retrieved relevant instances over the total number of retrieved instances. Recall is the ratio of retrieved relevant instances over the total number of relevant instances.

As shown above, Decision Tree Classifier ended up being significantly more accurate both in its Accuracy and its F1 Score.

DT Accuracy: 0.865 | KNN Accuracy: 0.791 | DT is 7.5% more accurate.

DT F1 Score: 0.844 | KNN F1 Score: 0.750 | DT has an F1 Score that was 9.4% higher.

I believe this is because Decision Tree is especially effective with simple relationships and datasets with both numerical and categorical data, whereas KNN tends to focus too much on outlier data and has difficulty deciding which features to measure distance to.
